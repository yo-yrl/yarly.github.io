152. Maximum Product Subarray

Posted on 2020-04-20

题目

Given an integer array nums, find the contiguous subarray within an array (containing at least one number) which has the largest product.

Example 1:

Input: [2,3,-2,4]
Output: 6
Explanation: [2,3] has the largest product 6.

Example 2:

Input: [-2,0,-1]
Output: 0
Explanation: The result cannot be 2, because [-2,-1] is not a subarray.

题意分析

  • 求一个连续子序列，且改序列的所有值乘积最大。【啊，一开始看成求最大和，哈哈哈哈product有乘积的意思(ﾉ”◑ ◑)ﾉ”(｡•́︿•̀｡)

解题思路

  • 和求最大和的连续子序列不同，在本体中会出现 负数 和 0 ， 负数会使得 当前子序列的值变成最小或最大，0则会变成0，当数列中存在偶数个负数时，乘上当前负数是对的选择，如果是奇数个，则需要额外的判断。
  • 暴力破解法。求所有的子序列，然后比较谁的乘积最大。肯定超时的写法。$O(n^2)$时间复杂度。
  • 动态规划。空间换时间。其实最大的难点是遇到负数时，我们不知道该不该要这个负数，因为此时它可能使得子序列乘积最小，但是，如果后头再遇到负数，则会得到乘积最大。 所以，我们可以用数组维护两个值，一个是当前子序列的最小值 minv[i]（出现0，或负数情况的记录） ，另一个是当前子序列的最大值 maxv[i], （0—i的子序列），且 res[i] = max( max[i-1]*nums[i], min[i-1] * nums[i] , nums[i])。

160. Intersection of Two Linked Lists

Posted on 2020-04-19

题目

Write a program to find the node at which the intersection of two singly linked lists begins.

Example 1:

Input: intersectVal = 8, listA = [4,1,8,4,5], listB = [5,0,1,8,4,5], skipA = 2, skipB = 3
Output: Reference of the node with value = 8
Input Explanation: The intersected node's value is 8 (note that this must not be 0 if the two lists intersect). From the head of A, it reads as [4,1,8,4,5]. From the head of B, it reads as [5,0,1,8,4,5]. There are 2 nodes before the intersected node in A; There are 3 nodes before the intersected node in B.

Notes:

  • If the two linked lists have no intersection at all, return null.
  • The linked lists must retain their original structure after the function returns.
  • You may assume there are no cycles anywhere in the entire linked structure.
  • Your code should preferably run in O(n) time and use only O(1) memory.

题意分析

  • 给定两个单向链表，返回他们汇聚的第一个节点，如果没有汇聚点，即单独的两个链表，则返回 nullptr.
  • 注意题目要求O(n)时间复杂度，O(1)的空间复杂度。

    解题思路

  • 如果不限制时间或空间复杂度，可以采用暴力破解的方法，去逐个遍历两个链表中是否存在两个相同的节点(O(m*n), O(1))。或者分别遍历两个链表，用map存储每个节点，如果存在重复节点，则是第一个汇聚点（O(m+n), O(n)）。
  • 我们可以分别确定两个链表的长度，a , b; 假设 a > b， 则链表a可先走(a-b)步，然后再同步遍历两个链表，则可以很方便找到汇聚点。a < b 是同理。【O(n)时间复杂度，O(1)的空间复杂度】

148. Sort List

Posted on 2020-04-19

题目

Sort a linked list in O(n log n) time using constant space complexity.

Example 1:

Input: 4->2->1->3
Output: 1->2->3->4

Example 2:

Input: -1->5->3->4->0
Output: -1->0->3->4->5

题意分析

给定一组数据(链表)，对链表进行排序。

解题思路

  • 重构链表：可以链表中的值都取出来，然后对数组进行排序，根据排序结果重新构建链表，或对原链表的值重新赋值。【优点：实现简单，缺点：需要额外空间】
  • 归并排序:对于两个给定的有序链表，我们很容器对其进行按序合并。我们可以考虑用归并排序的方法对链表进行排序：将链表均分为两个子链表，然后对两个子链表进行排序（依然是链表排序问题）得到有序的两个子链表，最终进行合并。

146. LRU Cache

Posted on 2020-04-19

题目

Design and implement a data structure for Least Recently Used (LRU) cache. It should support the following operations: get and put.

get(key) - Get the value (will always be positive) of the key if the key exists in the cache, otherwise return -1.
put(key, value) - Set or insert the value if the key is not already present. When the cache reached its capacity, it should invalidate the least recently used item before inserting a new item.

The cache is initialized with a positive capacity.

Follow up:

Could you do both operations in O(1) time complexity?

Example:

LRUCache cache = new LRUCache( 2 /* capacity */ );

cache.put(1, 1);
cache.put(2, 2);
cache.get(1);       // returns 1
cache.put(3, 3);    // evicts key 2
cache.get(2);       // returns -1 (not found)
cache.put(4, 4);    // evicts key 1
cache.get(1);       // returns -1 (not found)
cache.get(3);       // returns 3
cache.get(4);       // returns 4

题意分析

  • 实现LRU（最近最少使用）缓存机制的两个操作: get(key) 和 put(key, value)，即访问操作和更新操作。

解题思路

  • 对于访问操作，如果 key 存在，则返回 value , 否则返回 -1 。 由题可知，我们存入缓存中的数据是 key-value 数对，即key是唯一的。很容易由此联想到用map来存储这个数对，则 get(key)操作就是一次map.find()操作。

  • 对于更新操作，我们需要记录每个数据的最新访问时间和访问次数，当 cache 空间不足时，根据访问时间最小且访问次数最少 进行 数据替换。为此，我们需要为每个数据对 维护一个时间戳 和 使用次数。每次进行访问或更新操作时，需要对所有数据的时间戳进行加1。更新的数据时间戳设为0, 使用次数为1. 【不过这种方法每次都要遍历一次所有数据，很耗时】

  • 我们可以用链表（list）来存数据，进行访问或更新操作时，将相应数据放到链表头，则链表尾部存储的永远是可以替换出去的数据。

    • Tips:利用 list 的迭代器特性（删除或插入操作不会使迭代器失效！！！）， 我们可以再用一个map存储 key 及其 在链表中的位置，进行更新时就可以不用遍历链表了。

141. Linked List Cycle

Posted on 2020-04-16

题目

Given a linked list, determine if it has a cycle in it.

To represent a cycle in the given linked list, we use an integer pos which represents the position (0-indexed) in the linked list where tail connects to. If pos is -1, then there is no cycle in the linked list.

Example 1:

Input: head = [3,2,0,-4], pos = 1
Output: true
Explanation: There is a cycle in the linked list, where tail connects to the second node.

Follow up:

Can you solve it using O(1) (i.e. constant) memory?

题意分析

给定一个单向链表，判断是否有环。输入是一个数组，里头是节点的值，pos 是表示环开始的位置，如果没有环，则 pos = -1 .

求解思路

  • 判断一个节点是否重复访问。如果有环，肯定存在重复访问同一个节点的情况。我们可以用 set 保存每个节点，然后遍历链表。时间：$O(n)$, 空间： $O(n)$。

  • 快慢指针法。上述方法在链表很长的情况下会占用很多空间（ set ）。在一个单项链表中，存在两个指针对改链表进行遍历，一个指针每次走一步，另一个指针每次走两步，如果存在环，则慢指针 会 和 快指针 重合。如果不存在环，则会走到链表结尾（ nullptr ）。时间：$O(n)$， 空间：$O(1)$。这个指针的代码写得是真的很心累啊呜呜呜一定要小心

139. Word Break

Posted on 2020-04-14

题目

Given a non-empty string s and a dictionary wordDict containing a list of non-empty words, determine if s can be segmented into a space-separated sequence of one or more dictionary words.

Note:
The same word in the dictionary may be reused multiple times in the segmentation.
You may assume the dictionary does not contain duplicate words.

Example 1:

Input: s = "leetcode", wordDict = ["leet", "code"]
Output: true
Explanation: Return true because "leetcode" can be segmented as "leet code".

Example 2:

Input: s = "applepenapple", wordDict = ["apple", "pen"]
Output: true
Explanation: Return true because "applepenapple" can be segmented as "apple pen apple". *Note that* you are allowed to reuse a dictionary word.

题意分析

  • 题目给定一个非空字符串，和一个字符串数组，问字符串是否能根据数组内的小字符串进行分割~
  • 对于一个字符串s, 从 i = 0 位置开始，如果 s(i, j) 这个子串在数组内，则问题可以看做判断子串 s(j, end) 是否能进行分割的一个问题，我们可以考虑用深搜的方法进行求解

    解题思路

  • 用 set 记录数组内的小字符串，便于查找
  • 深搜求解，记录一个字符串的当前起始位置 sidx ，然后遍历 sidx – s.size()的位置 j , 当子串s.substr(sidx, j-sidx+1) 在 set 内， 则可以 对子问题进行求解。如果子问题返回 true ， 代表该字符串可以求解。
  • 深搜求法会出现很多重复计算的步骤，则我们可以一个数组来记录 当前开始位置为 sidx 的子字符串是否可以进行分割。
  • 动态规划方法。对于结束位置为i的字符串，dp[i]判断其是否可以分割————遍历0–i-1 位置 j , 如果 dp[j] == true （即可以分割）且 s.substr(j, i-j) 在 set 内， 则代表该字符串可以分割 –> dp[i] = true;

138. Copy List with Random Pointer

Posted on 2020-04-12

题目

A linked list is given such that each node contains an additional random pointer which could point to any node in the list or null.

Return a deep copy of the list.

The Linked List is represented in the input/output as a list of n nodes. Each node is represented as a pair of [val, random_index] where:

  • val: an integer representing Node.val
  • random_index: the index of the node (range from 0 to n-1) where random pointer points to, or null if it does not point to any node.

题意分析

给定一个链表，除了 next 指针，还有一个 random 指针，指向链表中其他节点或 nullptr 。我们要对这个链表进行深拷贝，并返回新链表。

因为新旧链表的地址不一样，我们不能通过简单的 p == p1 寻找新链表节点的 random 指针。所以我们需要想办法去解决这个新旧节点地址不同的问题。

解题思路

  • 用map进行新旧节点的映射。这样在寻找random指针时就很方便~ 遍历一遍链表，创建新链表并完成map映射，再进行一遍遍历，完成random指针的指向~ 【时间复杂度:O(n), 空间复杂度: O(n)】
  • 在原有链表上增加新节点：old1 -> new1 -> old2 -> new2，然后 新节点的Random指针就是旧节点的random指针的next；最后，将链表拆开成新旧链表，返回新链表【时间复杂度：O(n), 空间复杂度：O(1)】

136. Single Number

Posted on 2020-04-11

题目

Given a non-empty array of integers, every element appears twice except for one. Find that single one.

Note:

Your algorithm should have a linear runtime complexity. Could you implement it without using extra memory?

Example 1:

Input: [2,2,1]
Output: 1

Example 2:

Input: [4,1,2,1,2]
Output: 4

题意分析

给定一个非空整数数组，其中只有一个数字只出现一次，其他数字都出现两次，先要求找出该只出现一次的数字。

解题思路

  • 要求o(n)复杂度，一般想到就是空间换时间。可以通过记录整数出现次数来找出答案，用map来存储 value-count 数对，两次遍历解决：第一次遍历数组，记录 value-count， 第二次，则判断整数的count是否为1。
  • 利用异或解决，a ^ a = 0, 将数组每个值进行异或，则最后的值肯定是 只出现一次的那个数字。
