238. Product of Array Except Self

Posted on 2020-05-05

题目

Given an array nums of n integers where n > 1, return an array output such that output[i] is equal to the product of all the elements of nums except nums[i].

Example:

Input:  [1,2,3,4]
Output: [24,12,8,6]

Constraint: It’s guaranteed that the product of the elements of any prefix or suffix of the array (including the whole array) fits in a 32 bit integer.

Note: Please solve it without division and in O(n).

Follow up:

  • Could you solve it with constant space complexity? (The output array does not count as extra space for the purpose of space complexity analysis.)

题意分析

给定一个数组 nums ，返回另一个数组 res ，其中 res[i] 为除 nums[i]之外所有值得乘积。不准用除法。

解题思路

  • 分别得到位置i的左边数乘积和右边数乘积，然后求取最终结果。用left和right分别存储乘积。例如：对于数组[1,2,3,4], left = [1,1,2,6], right = [24,12,4,1], 则 res = [24, 12, 8, 6].
  • 节省空间的方法：减掉right和left的额外空间。直接存在res中。

237. Delete Node in a Linked List

Posted on 2020-05-05

题目

Write a function to delete a node (except the tail) in a singly linked list, given only access to that node.

Example 1:

Input: head = [4,5,1,9], node = 5
Output: [4,1,9]
Explanation: You are given the second node with value 5, the linked list should become 4 -> 1 -> 9 after calling your function.

Note:

  • The linked list will have at least two elements.
  • All of the nodes’ values will be unique.
  • The given node will not be the tail and it will always be a valid node of the linked list.
  • Do not return anything from your function.

题意分析

给定一个链表及其一个节点，删掉该节点。

输入约束：1-删除节点肯定存在，且不是尾节点 2-链表中没有重复值得节点 3-链表存在2个或2个以上节点。

解题思路

  • 由于给出函数中只有要删除的节点这一个参数，故而我们不能从头遍历链表进行节点删除。(我还以为我不小心把函数参数删了，自己还补上了head参数 = =)
  • 只给出要删除的节点，我们可以 将其下一个节点拷贝给当前节点，然后删除下一个节点来完成删除功能，且不破坏链表结构。比如： 1->2->3; 删除2的话，则 1-拷贝： 1->3->null 单独节点；3 ; 2-删除：单独节点3。

236. Lowest Common Ancestor of a Binary Tree

Posted on 2020-05-05

题目

Given a binary tree, find the lowest common ancestor (LCA) of two given nodes in the tree.

According to the definition of LCA on Wikipedia: “The lowest common ancestor is defined between two nodes p and q as the lowest node in T that has both p and q as descendants (where we allow a node to be a descendant of itself).”

Example 1:

Input: root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 1
Output: 3
Explanation: The LCA of nodes 5 and 1 is 3.

Note:

  • All of the nodes’ values will be unique.
  • p and q are different and both values will exist in the binary tree.

题意分析

给定一个二叉树，和其中2个节点，求2个节点的最低公共父节点。

解题思路

  • 节点路径。分别求取两个节点的路径（从根节点到该节点），然后比较两条路径，得到最低公共父节点。时间：$O(n)$, 空间：$O(n)$。
  • 还有一种递归遍历方法，代码很简洁，感觉会节省空间，但还没看明白。

234. Palindrome Linked List

Posted on 2020-05-04

题目

Given a singly linked list, determine if it is a palindrome.

Example 1:

Input: 1->2
Output: false

Example 2:

Input: 1->2->2->1
Output: true

Follow up:

Could you do it in O(n) time and O(1) space?

题意分析

给定一个链表，判断是否是回文链表。

回文串：正着看和反着看是一样的，比如a, aa, aba等

解题思路

  • 需要有一个链表的倒序存在，用来判断与其正序是否相同，很容易我们就能想到stack这个数据结构，利用其先进后出的特性来得到链表的倒序。时间：$O(n)$, 空间：$O(n)$
  • follow up要求空间为O(1)， 则我们不能利用stack来保存值。所以我们得用别的办法得到链表的倒序。由此可以想到反转链表这种方法，将链表反转过来。但是，对整个链表进行反转是行不通的（因为链表只有一条，又不能copy一个新的）；我们还得再想想——可以将链表二分，对一半链表进行倒序，然后对比两条链表的值是否相等。此中用到的方法：1- 寻找链表中间节点（对半分链表） 2- 反转链表。

227. Basic Calculator II

Posted on 2020-05-03

题目

Implement a basic calculator to evaluate a simple expression string.

The expression string contains only non-negative integers, +, -, *, / operators and empty spaces . The integer division should truncate toward zero.

Example 1:

Input: "3+2*2"
Output: 7

Example 2:

Input: " 3/2 "
Output: 1

Example 3:

Input: " 3+5 / 2 "
Output: 5

Note:

You may assume that the given expression is always valid.
Do not use the eval built-in library function.

题意分析

给定一个字符串，是一个计算式子，包含数字和 加减乘除四种运算符号，以及空格。我们返回计算式子的最终值。

解题思路

  • 一开始想到用栈来分别保存 数值 和 运算符。遍历字符串，遇到空格就略过；遇到数值就存储起来；对于运算符，乘法除法的优先级高于加法减法，我的思路是遇到乘除法就先进行计算，最后将式子变为只剩下加减法（平级）：如果遇到乘法除法，则取其前后两个数值进行计算再将 结果数值存储起来；遇到加法、减法符号就直接入栈。
  • 后来发现，用栈存储是不对的，因为加减法是平级的，我们需要从头往后计算，而不是先计算后头的。出错例子： “1-1+1”， 如果用栈存储，栈内数据分别为：1, 1, 1 和 -, +；先计算加法，栈内数据更新为： 1,2 和 -；然后最后结果是-1，出错。所以应该是从头开始计算。用vector容器存储就可以。

217. Contains Duplicate

Posted on 2020-05-03

题目

Given an array of integers, find if the array contains any duplicates.

Your function should return true if any value appears at least twice in the array, and it should return false if every element is distinct.

Example 1:

Input: [1,2,3,1]
Output: true

Example 2:

Input: [1,2,3,4]
Output: false

Example 3:

Input: [1,1,1,3,3,4,3,2,4,2]
Output: true

题意分析

给定一个数组，如果出现重复值，返回true， 否则返回false。

解题思路

  • 暴力破解。将每个数与其后头的数进行比较。时间：$O(n^2)$.
  • 排序。对数组进行排序，然后从头到尾遍历一遍。重复值总在相邻的位置。时间：$O(nlog(n))$.
  • set。用set保存数组中的值，由于其不能保存重复值，如果set的大小 比 数组大小小，则出现重复值。空间：$O(n)$.

215. Kth Largest Element in an Array

Posted on 2020-05-01

题目

Find the kth largest element in an unsorted array. Note that it is the kth largest element in the sorted order, not the kth distinct element.

Example 1:

Input: [3,2,1,5,6,4] and k = 2
Output: 5

Example 2:

Input: [3,2,3,1,2,4,5,5,6] and k = 4
Output: 4

Note:

  • You may assume k is always valid, 1 ≤ k ≤ array’s length.

    题意分析

给定一个数组，返回第k大的数。

比如 3,2,1,5,6,4里头，第k = 2大的数是 5 ， 则返回 5 。

解题思路

  • 对数组进行排序。然后返回第k大值。排序算法时间复杂度：$O(nlog(n))$.
  • 最小堆。维护一个k个值的最小堆，堆顶是最小值，其他值比堆顶都大。用数组前k个值构建最小堆，然后对其后n-k个值进行遍历，当 值比堆顶大时，则调整堆（删除堆顶元素, 然后压入新值并调整堆）。时间复杂度：$O(n)$, 空间复杂度：$O(k)$.
  • 快排思想。快排的思路是，确定一个哨兵v，比v值小的的排在哨兵左边，反之则排在右边，使得整个序列局部有序，接下来分别对两边进行排序。在这个过程当中可以确定的是，此时哨兵v所在的位置 是 完成排序后的位置，即v值已完成排序。利用这一特性，我们可以通过判断哨兵v所在位置是否为k来返回我们要寻找的第k大的值。时间复杂度：$O(n)$，不知道怎么证。缺点：对数组进行修改。

207. Course Schedule

Posted on 2020-05-01

题目

There are a total of numCourses courses you have to take, labeled from 0 to numCourses-1.

Some courses may have prerequisites, for example to take course 0 you have to first take course 1, which is expressed as a pair: [0,1]

Given the total number of courses and a list of prerequisite pairs, is it possible for you to finish all courses?

Example 1:

Input: numCourses = 2, prerequisites = [[1,0]]
Output: true
Explanation: There are a total of 2 courses to take. 
            To take course 1 you should have finished course 0. So it is possible.

Example 2:

Input: numCourses = 2, prerequisites = [[1,0],[0,1]]
Output: false
Explanation: There are a total of 2 courses to take. 
            To take course 1 you should have finished course 0, and to take course 0 you should
            also have finished course 1. So it is impossible.

Constraints:

  • The input prerequisites is a graph represented by a list of edges, not adjacency matrices. Read more about how a graph is represented.
  • You may assume that there are no duplicate edges in the input prerequisites.
  • 1 <= numCourses <= 10^5

题意分析

选课开始了，有一些课程进修之前必须学过其他课程，即先导课程。举个例子，我们大一一般上的是基础课程，如C++语言程序设计，高数等；等到大二大三才上专业课程。我们必须先打好基础，才能往下继续学习。

现给定课程及其先导课程，让我们判断按照输入是否能够成功修完所有课。

解题思路

  • 第一想法是有向图，判断是否有环。如果出现环，则不能选秀完所有课程。
  • emm，问题来了，咋写代码呢？了解到该题目是经典的拓扑排序问题，所以先去复习概念。然后再借鉴其他人的思路完成代码。
  • 主要实现算法有两种：广度优先搜索 和 深度优先搜索
    1. 图的构建：邻接矩阵，邻接表等。如果一门课程有先导课程，则可用入度+1表示.
    2. BFS. 根据节点的入度进行判断课程学习顺序。假设一门课程i入度为0，则可以确定该课程肯定是其他课程的先导课或者完全无关的，即可以先进行学习。一旦遍历过该课程，则将其直接影响课程的入度-1。重复以上动作直至所有课程选修完成。如果没有完成，则表示程序结束了，存在课程入度>0，即存在环.
    3. DFS. 遍历有向图，如果出现对某个节点的重复访问，则出现环。
