300. Longest Increasing Subsequence

Posted on 2020-05-09

题目

Given an unsorted array of integers, find the length of longest increasing subsequence.

Example:

Input: [10,9,2,5,3,7,101,18]
Output: 4 
Explanation: The longest increasing subsequence is [2,3,7,101], therefore the length is 4.

Note:

There may be more than one LIS combination, it is only necessary for you to return the length.

Your algorithm should run in O(n2) complexity.

Follow up:

Could you improve it to O(n log n) time complexity?

题意分析

给定一个数组，求该数值中最长的不连续递增序列的长度。

解题思路

  • 想不到什么暴力破解的思路。如果有，那估计是以每个位置为开头，求出其与之后数字的所有排列组合，然后再一一对比。啊，好累。

  • 动态规划。对每个数字，以其为结尾，求出其与前头数组组成的最长递增子序列。递归式子如下：

    $dp[i] = max(dp[i], 1+dp[j])$, where $j < i$ and $nums[j] < nums[i]$.

    时间复杂度为：$O(n^2)$.因为对于每个数i，需要和前头i-1个数进行比较。

315. Count of Smaller Numbers After Self

Posted on 2020-05-09

题目

You are given an integer array nums and you have to return a new counts array. The counts array has the property where counts[i] is the number of smaller elements to the right of nums[i].

Example:

Input: [5,2,6,1]
Output: [2,1,1,0] 
Explanation:
To the right of 5 there are 2 smaller elements (2 and 1).
To the right of 2 there is only 1 smaller element (1).
To the right of 6 there is 1 smaller element (1).
To the right of 1 there is 0 smaller element.

题意分析

给定一个数组，找出每一个数，在其之后比它小的数的个数。

解题思路

  • 暴力破解。对于每个数，都寻找一遍其后头的数一遍。
  • 插入排序算法。前提条件：题目没有限定不可以改变输入数组。我们想的是能尽快得到比这个数小的数值，可以利用一个有序性。而插入排序和这种想法很像，就是寻找这个数的位置，借此我们可以知道这个数在之前数组中的位置（这就相当于知道了它前头有多少个比它小的数！）这种思路，也是$O(n^2)$的时间复杂度。

283. Move Zeroes

Posted on 2020-05-07

题目

Given an array nums, write a function to move all 0’s to the end of it while maintaining the relative order of the non-zero elements.

Example:

Input: [0,1,0,3,12]
Output: [1,3,12,0,0]

Note:

  • You must do this in-place without making a copy of the array.
  • Minimize the total number of operations.

题意分析

给定一个数组，要求我们不能增加额外的空间，将数组中 的 0 值交换到数组最后，并且保持非0数组间的相对顺序（类似稳定排序的概念）。

解题思路

  • 两个指针。看到需要在原数组上对两类值进行交换位置，就考虑到了用双指针：一个指针a用于指向一类值的位置，另一个b指向另一类值的位置，由此交换两个位置的值，并且更新指针指向位置，循环致结束。
  • 保持相对顺序。一开始，就想着双指针分别从两头开始寻找对应位置，然后交换值。但是结果显示，这样做会破坏各个元素的相对位置。比如 0,1,0,3,12 的结果会是 12, 1, 3, 0, 0. 这样是不对的。所以，我们得注意替换过程中不能改变相对位置。最后，可以将连个指针都从 0 位置开始寻找，这样交换时，肯定能保存非0数值的相对位置： 比如，0,1,0,3,12, 第一次：1,0,0,3,12,第二次：1,3,0,0,12,第三次：1,3,12,0,0.

279. Perfect Squares

Posted on 2020-05-07

题目

Given a positive integer n, find the least number of perfect square numbers (for example, 1, 4, 9, 16, …) which sum to n.

Example 1:

Input: n = 12
Output: 3 
Explanation: 12 = 4 + 4 + 4.

Example 2:

Input: n = 13
Output: 2
Explanation: 13 = 4 + 9.

题意分析

给定一个数，求解其最少由几个平方数值组成。平方数值：1,4,9,16,…。

解题思路

  • 啊，一开始真的摸不着头脑。只能看别人的blog汲取智慧。

  • 最短路径（广搜）。

    举个例子，该题可以看做是n 到 0 的不同路径组成的图，求解其最短路径。源是 n， 终点是0， 中间的节点值有 1–n-1。每条路径上，假设当前点是n_i, 则下个点n_i+1则有是 n_i - (平方数值)，有k中可能。我们就是想找出源节点到目的节点的最短路径捏。

    最短路径 ：从源节点出发，遍历平方数值（广搜），得到下一个可以到达的节点，判断是否到达0。如果没有节点是0，则继续上一步骤，遍历下一波节点。每次向前走一步，我们就可以将步数+1，最后到达0节点是，返回步数。

  • 动态规划。

    由BFS的陈述中，可以发现该题还可以用动态规划求解，其推导公式是：$dp[i] = min(dp[i], 1+ dp[nums[i]- squaresNum[j]])$.

240. Search a 2D Matrix II

Posted on 2020-05-06

题目

Write an efficient algorithm that searches for a value in an m x n matrix. This matrix has the following properties:

Integers in each row are sorted in ascending from left to right.
Integers in each column are sorted in ascending from top to bottom.

Example:

Consider the following matrix:

[
[1,   4,  7, 11, 15],
[2,   5,  8, 12, 19],
[3,   6,  9, 16, 22],
[10, 13, 14, 17, 24],
[18, 21, 23, 26, 30]
]
Given target = 5, return true.

Given target = 20, return false.

题意分析

  • 给定一个矩阵，和一个数，找出矩阵中是否存在这个数
  • 矩阵存在局部有序特性！每一行，从左到右递增；每一列，从上到下递增。

解题思路

  • 暴力破解。遍历矩阵，查找是否存在该数值。
  • 利用有序性缩减查找范围。我们可以发现，每一行，最后一个值是该行最大值，该列最小值。如果将该数与目标值进行比较，根据大小，我们可以直接排除一行或一列的比较次数。比如查找5， 与 15比较，则排除最后一列，然后与11比较，排除该列，与7比较，排除这一列；与4比较，排除该行，与5比较，get！

239. Sliding Window Maximum

Posted on 2020-05-06

题目

Given an array nums, there is a sliding window of size k which is moving from the very left of the array to the very right.

You can only see the k numbers in the window. Each time the sliding window moves right by one position. Return the max sliding window.

Follow up:

Could you solve it in linear time?

Example:

Input: nums = [1,3,-1,-3,5,3,6,7], and k = 3
Output: [3,3,5,5,6,7] 
Explanation: 

Window position                Max
---------------               -----
[1  3  -1] -3  5  3  6  7       3
1 [3  -1  -3] 5  3  6  7       3
1  3 [-1  -3  5] 3  6  7       5
1  3  -1 [-3  5  3] 6  7       5
1  3  -1  -3 [5  3  6] 7       6
1  3  -1  -3  5 [3  6  7]      7

Constraints:

1 <= nums.length <= 10^5
-10^4 <= nums[i] <= 10^4
1 <= k <= nums.length

题意分析

给定一个数组，和一个滑动窗口大小，求数组中，每个滑动窗口中的最大值。

解题思路

  • 暴力。维护滑动窗口的left_idx 和 right_idx ，每次对窗口内的值求最大值。O(n*k).
  • 利用数据结构存储这个窗口的最大值，避免每次都求最大值。先考虑，对于一个窗口内的 1 3 -1 值，在顺序访问时，用啥数据结构在线性时间内得到最大值。容易想到的是队列，队列中先存 1， 然后访问到3时，3比1大，则将1弹出，存入3； 访问-1，比3小，则存在3后头，此时队列为 3,-1； 队头就是这个窗口内的最大值。当窗口滑动时，继续这样比较，我们就可以很容易得到每个窗口的最大值了。note1： 存入位置大的较小值，比如-1 是 因为 当3不在窗口内时，-1可能是下个窗口的最大值，所以需要存入-1。note2：存入队列的应该是数值的下标，这是为了判断当前队头是否不在窗口内。

268. Missing Number

Posted on 2020-05-06

题目

Given an array containing n distinct numbers taken from 0, 1, 2, …, n, find the one that is missing from the array.

Example 1:

Input: [3,0,1]
Output: 2

Example 2:

Input: [9,6,4,2,3,5,7,0,1]
Output: 8

Note:

Your algorithm should run in linear runtime complexity. Could you implement it using only constant extra space complexity?

题意分析

题目给定长度为n的数组，存储的值范围在0–n。 0–n数值范围有n+1个数，需要我们找出数组中缺失的那个数。

解题思路

  • 排序+遍历。但题目要求 $O(n)$时间，这个是$O(nlogn)$.
  • hash。遍历一遍数组nums存入hash表,然后遍历0-n，看哪个数不存在。
  • 求和。分别求nums的和 以及 0–n的和，求两者差，就可以知道缺了哪个数了。

242. Valid Anagram

Posted on 2020-05-06

题目

Given two strings s and t , write a function to determine if t is an anagram of s.

Example 1:

Input: s = "anagram", t = "nagaram"
Output: true

Example 2:

Input: s = "rat", t = "car"
Output: false

Note:

You may assume the string contains only lowercase alphabets.

Follow up:

What if the inputs contain unicode characters? How would you adapt your solution to such case?

题意分析

给定2个字符串 s, t, 判断 t 是否是 s 的一个变形。意思是他两是否由同样个数的字符组成。

解题思路

  • hash。可以用hash表来分别统计字符串 s, t 中出现的字符次数，然后进行比较。 因为题目限定为小写字母组成字符串，则我们可以用长度为26的数组构成hash表，空间复杂度为$O(1)$。如果是有unicode组成的字符串，同理，一共存在200多个（256？）Unicode字符，可以用长度为256的数组构成hash表。

238. Product of Array Except Self

Posted on 2020-05-05

题目

Given an array nums of n integers where n > 1, return an array output such that output[i] is equal to the product of all the elements of nums except nums[i].

Example:

Input:  [1,2,3,4]
Output: [24,12,8,6]

Constraint: It’s guaranteed that the product of the elements of any prefix or suffix of the array (including the whole array) fits in a 32 bit integer.

Note: Please solve it without division and in O(n).

Follow up:

  • Could you solve it with constant space complexity? (The output array does not count as extra space for the purpose of space complexity analysis.)

题意分析

给定一个数组 nums ，返回另一个数组 res ，其中 res[i] 为除 nums[i]之外所有值得乘积。不准用除法。

解题思路

  • 分别得到位置i的左边数乘积和右边数乘积，然后求取最终结果。用left和right分别存储乘积。例如：对于数组[1,2,3,4], left = [1,1,2,6], right = [24,12,4,1], 则 res = [24, 12, 8, 6].
  • 节省空间的方法：减掉right和left的额外空间。直接存在res中。

237. Delete Node in a Linked List

Posted on 2020-05-05

题目

Write a function to delete a node (except the tail) in a singly linked list, given only access to that node.

Example 1:

Input: head = [4,5,1,9], node = 5
Output: [4,1,9]
Explanation: You are given the second node with value 5, the linked list should become 4 -> 1 -> 9 after calling your function.

Note:

  • The linked list will have at least two elements.
  • All of the nodes’ values will be unique.
  • The given node will not be the tail and it will always be a valid node of the linked list.
  • Do not return anything from your function.

题意分析

给定一个链表及其一个节点，删掉该节点。

输入约束：1-删除节点肯定存在，且不是尾节点 2-链表中没有重复值得节点 3-链表存在2个或2个以上节点。

解题思路

  • 由于给出函数中只有要删除的节点这一个参数，故而我们不能从头遍历链表进行节点删除。(我还以为我不小心把函数参数删了，自己还补上了head参数 = =)
  • 只给出要删除的节点，我们可以 将其下一个节点拷贝给当前节点，然后删除下一个节点来完成删除功能，且不破坏链表结构。比如： 1->2->3; 删除2的话，则 1-拷贝： 1->3->null 单独节点；3 ; 2-删除：单独节点3。
