215. Kth Largest Element in an Array

Posted on 2020-05-01

题目

Find the kth largest element in an unsorted array. Note that it is the kth largest element in the sorted order, not the kth distinct element.

Example 1:

Input: [3,2,1,5,6,4] and k = 2
Output: 5

Example 2:

Input: [3,2,3,1,2,4,5,5,6] and k = 4
Output: 4

Note:

  • You may assume k is always valid, 1 ≤ k ≤ array’s length.

    题意分析

给定一个数组，返回第k大的数。

比如 3,2,1,5,6,4里头，第k = 2大的数是 5 ， 则返回 5 。

解题思路

  • 对数组进行排序。然后返回第k大值。排序算法时间复杂度：$O(nlog(n))$.
  • 最小堆。维护一个k个值的最小堆，堆顶是最小值，其他值比堆顶都大。用数组前k个值构建最小堆，然后对其后n-k个值进行遍历，当 值比堆顶大时，则调整堆（删除堆顶元素, 然后压入新值并调整堆）。时间复杂度：$O(n)$, 空间复杂度：$O(k)$.
  • 快排思想。快排的思路是，确定一个哨兵v，比v值小的的排在哨兵左边，反之则排在右边，使得整个序列局部有序，接下来分别对两边进行排序。在这个过程当中可以确定的是，此时哨兵v所在的位置 是 完成排序后的位置，即v值已完成排序。利用这一特性，我们可以通过判断哨兵v所在位置是否为k来返回我们要寻找的第k大的值。时间复杂度：$O(n)$，不知道怎么证。缺点：对数组进行修改。

207. Course Schedule

Posted on 2020-05-01

题目

There are a total of numCourses courses you have to take, labeled from 0 to numCourses-1.

Some courses may have prerequisites, for example to take course 0 you have to first take course 1, which is expressed as a pair: [0,1]

Given the total number of courses and a list of prerequisite pairs, is it possible for you to finish all courses?

Example 1:

Input: numCourses = 2, prerequisites = [[1,0]]
Output: true
Explanation: There are a total of 2 courses to take. 
            To take course 1 you should have finished course 0. So it is possible.

Example 2:

Input: numCourses = 2, prerequisites = [[1,0],[0,1]]
Output: false
Explanation: There are a total of 2 courses to take. 
            To take course 1 you should have finished course 0, and to take course 0 you should
            also have finished course 1. So it is impossible.

Constraints:

  • The input prerequisites is a graph represented by a list of edges, not adjacency matrices. Read more about how a graph is represented.
  • You may assume that there are no duplicate edges in the input prerequisites.
  • 1 <= numCourses <= 10^5

题意分析

选课开始了，有一些课程进修之前必须学过其他课程，即先导课程。举个例子，我们大一一般上的是基础课程，如C++语言程序设计，高数等；等到大二大三才上专业课程。我们必须先打好基础，才能往下继续学习。

现给定课程及其先导课程，让我们判断按照输入是否能够成功修完所有课。

解题思路

  • 第一想法是有向图，判断是否有环。如果出现环，则不能选秀完所有课程。
  • emm，问题来了，咋写代码呢？了解到该题目是经典的拓扑排序问题，所以先去复习概念。然后再借鉴其他人的思路完成代码。
  • 主要实现算法有两种：广度优先搜索 和 深度优先搜索
    1. 图的构建：邻接矩阵，邻接表等。如果一门课程有先导课程，则可用入度+1表示.
    2. BFS. 根据节点的入度进行判断课程学习顺序。假设一门课程i入度为0，则可以确定该课程肯定是其他课程的先导课或者完全无关的，即可以先进行学习。一旦遍历过该课程，则将其直接影响课程的入度-1。重复以上动作直至所有课程选修完成。如果没有完成，则表示程序结束了，存在课程入度>0，即存在环.
    3. DFS. 遍历有向图，如果出现对某个节点的重复访问，则出现环。

206. Reverse Linked List

Posted on 2020-04-27

题目

Reverse a singly linked list.

Example:

Input: 1->2->3->4->5->NULL
Output: 5->4->3->2->1->NULL

Follow up:

A linked list can be reversed either iteratively or recursively. Could you implement both?

题意分析

给定一个链表，实现链表翻转功能。

解题思路

  • 用栈存储节点信息。遍历链表，用栈存储每个节点的值。再遍历一遍，将链表的值替换成栈顶的值，并出栈。
  • 三指针方法。用三个指针 *pre, *cur, *next 分别记录当前遍历节点，及其前后两个节点，便于将指针翻转过来。翻转过程： 比如 1->2->3; pre = 1, cur = 2, next = 3; cur->next = pre; 1<-2->3; 然后使得pre = cur = 2; cur = next = 3, next = nullptr; 以此循环。
  • 递归方法。

204. Count Primes

Posted on 2020-04-27

题目

Count the number of prime numbers less than a non-negative number, n.

Example:

Input: 10
Output: 4
Explanation: There are 4 prime numbers less than 10, they are 2, 3, 5, 7.

题意分析

给定一个数，找出这个数范围的 质数 的个数。

解题思路

  • 对于质数来说，是 只能对1或其本身除尽。
  • 可以对每个数进行判断是否是质数，然后进行统计。
  • 对于每个质数（2，3，5，等），我们可以排除他的倍数不是质数。比如 2 可以排除 4,6,8,10等，则3是质数，又可以排除 6,9,12等；接下来4不是质数，5是，以此循环。

Those numbers for which this process ends in 1 are happy numbers.

Return True if n is a happy number, and False if not.

200. Number of Islands

Posted on 2020-04-27

题目

Given a 2d grid map of ‘1’s (land) and ‘0’s (water), count the number of islands. An island is surrounded by water and is formed by connecting adjacent lands horizontally or vertically. You may assume all four edges of the grid are all surrounded by water.

Example 1:

Input:
11110
11010
11000
00000

Output: 1

题意分析

给定一个由字符 ‘0’ 和 ‘1’组成的二维数组，0代表水，1代表陆地，求给定数组里头小岛有几个。小岛的定义是：四面被水包围的陆地，设边界处都是水。

解题思路

  • 深搜思路。遍历数组，当遇到一块陆地（字符’1’）时，则将该小岛标记起来（遍历每一个相邻的’1’），表示已经访问过该小岛。然后再继续寻找下一个小岛。时间：$O(n)$

198. House Robber

Posted on 2020-04-26

题目

You are a professional robber planning to rob houses along a street. Each house has a certain amount of money stashed, the only constraint stopping you from robbing each of them is that adjacent houses have security system connected and it will automatically contact the police if two adjacent houses were broken into on the same night.

Given a list of non-negative integers representing the amount of money of each house, determine the maximum amount of money you can rob tonight without alerting the police.

Example 1:

Input: [1,2,3,1]
Output: 4
Explanation: Rob house 1 (money = 1) and then rob house 3 (money = 3).
         Total amount you can rob = 1 + 3 = 4.

题意分析

有一个盗贼，想要入屋盗窃。现小区内有一排房子，房子都装有报警器，如果偷了两家相邻的房子，则会自动报警。假设小偷知道每家每户的可盗取金额，问小偷最多能偷多少钱。

解题思路

  • 动态规划方法。 对于当前i房子，他可以选择偷或不偷，如果偷， 他不能偷 i-1家的，下一次得从i-2家投起；如果不偷，则可以再从i-1家偷起。所以可以退出一个递推式子： $dp[i] = max(dp[i-2]+money[i], dp[i-1])$.
